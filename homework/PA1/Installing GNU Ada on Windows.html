Installing GNU Ada (GNAT) for Windows

The cygwin environment will be used for programming in several languages, including Ada, this semester.  It provides a linux-like environment for Windows users.  The GNAT compiler, based on GCC, will be used to compile Ada programs and is easily installed on a Windows machine.

To install cygwin on your machine, go to  http://www.cygwin.org  and click on "Install cygwin" on the top left hand side of the page.  Then click on  "setup-x86_64.exe" to download the setup program.  Once that setup program has been downloaded, run it.

Follow the instructions for installing cygwin, accepting the default settings until you get to the "Select Packages" window.  At that point, you will need to make sure that GCC gets installed. To do this:

  1. At the top of the window, in the drop-down menu labeled "View", choose "Category".
  2. Click on the "+" next to the "All" entry to make the list of available packages appear.
  3. Click on the "+" next to "Devel" to expand the list of available development tools.
  4. Scroll down to "gcc-ada" and, where you see the drop-down menu that says "skip", select "7.4.0-1" instead.
  5. Not far below, where it says "gcc-core",  select "7.4.0-1" from the drop-down menu (instead of "skip").

Then, click "next" at the bottom of the window and continue to click "next" when prompted until the installation finishes (at which point you'll click "Finish").  At the end of the installation process, it will offer to "Create icon on Desktop", which I recommend doing.

Once the installation process is finished, double-click on the cygwin icon.  A shell should open.  Type the command "which gnatmake" and make sure that it displays a path for the gnatmake utility.

Testing the compiler

In the Ada resources folder, click on "Simple Ada program to test compiler".  A zip file will be downloaded.  When you unzip it, you will see several files with .ads and .adb extensions.  In a cygwin shell, go to the directory containing the files. Then type,

gnatmake objmain.adb

You will see that gcc is invoked several times, along with gnatbind and gnatlink.

To run the program, type

./objmain

(you won't need the "./" if your PATH environment variable contains ".")

Note: In the cygwin shell, your home directory corresponds to the windows directory c:\cygwin64\home\username\, where username is whatever your username on the computer is.  You may want to copy the downloaded sample Ada files (with the .ads and .adb extensions) into that home directory before trying to compile them.
